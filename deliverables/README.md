# Deliverables — sample generated decks

These `.pptx` files are **sample outputs generated by this project** (one performance-review
deck per merchant from the provided dummy data), committed here so the generated presentation
is visible without running anything:

| File | Merchant | Profile |
|---|---|---|
| `acme_performance_review.pptx` | ACME | Post auth · Strategic (count + amount-weighted) |
| `cyberdyne_systems_performance_review.pptx` | Cyberdyne Systems | Pre auth · Strategic |
| `vandelay_industries_performance_review.pptx` | Vandelay Industries | Post auth · Enterprise (count only) |

Each deck: **Title → Executive Summary → one slide per KPI** (monthly trend + quarterly bars +
a short written analysis) **→ Notes & Methodology**, all sharing one theme.

## Regenerating

These are not hand-made — they are produced by the pipeline and can be regenerated any time:

```bash
make ingest    # build the DuckDB facts (no API key)
make decks     # generate fresh decks under data/output/ (needs ANTHROPIC_API_KEY)
make deliverables   # copy each merchant's latest deck here with a friendly name
```

Fresh, versioned outputs land under `data/output/decks/<merchant>/<merchant>_<version>.pptx`
(gitignored — one UTC version per run, full history kept). This folder holds a clean, committed
snapshot for review.
